• SAP Web IDE — Design Guidelines

Visual Design Foundations

Restrictions

As described in the Design Principles the Flat theme of the SAP Web IDE is flat and light, minimalistic and clutter-free. Its minimalism comes at its cost and one has to be careful not to be tempted to show one’s own feature in the best possible light by means of eye candy, or the powerful styling features of CSS3.

Use drop shadows and similar effects sparingly

The only elements that use a drop-shadow are pop-ups, dialogs, menu drop downs and WYSIWYG preview—elements that actually overlay the background area. These elements must not overlay each other. All of them use the same drop shadow style with 1 pixel spacing to the bottom and a spread of 2 pixels.

Do not make use of gradients

The only exception is the background image where a huge gradient is used to draw an abstract depiction of a horizon line with the Organic Trees.

Avoid bold font weights

Due to the flat and light design approach of the SAP Web IDE, the use of bold fonts is exceptional.

Animate only when meaningful

The SAP Web IDE hardly makes use of animations, fading, or CSS transforms. Animations for the menu (fading) or the menu (rotating and zooming) have been intentionally removed.

Avoid borders and underlines

Different areas are not differentiated by means of margings, borders, or drop shadows but by different color backgrounds. This not only de-clutters the user interfaces, in sum it also provides screen real estate for more important elements.
