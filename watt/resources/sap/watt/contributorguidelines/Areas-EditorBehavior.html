Editor Behavior

Each File has Several Representations With One Purpose each.

Tabs and their Representing Views

In the SAP Web IDE, opened files are represented as tabs in the main area. Depending on its type, each file can have various representations to their content. These representations are called Views.

For example, a certain XML files can be represented in Code view (which shows the actual XML code as text), in Design view (which shows the UI elements contained in the XML in an WYSIWYG interface), or as Data Binding view (which shows a view to connect UI elements to data sources). These Views are shown as additional tabs at the bottom of each file tab.

Links

To connect different files and views to each other, the SAP Web IDE makes use of Hyperlinks. Selecting a link in a View, the related file is opened in the desired Views.

Pop-Out / Dock-out

To show two views of one file side-by-side, currently not selected views can be popped-out to the side of the selected view via its contextual menu. The tab at the bottom then jointly displays both tabs. Popped-out views can be separated again via the contextual menu.
