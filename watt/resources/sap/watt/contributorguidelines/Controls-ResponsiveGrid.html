Responsive Grid

In the context of visual design, a grid is a structure consisting of a series of intersecting vertical and horizontal lines used to organize content. The grid serves as a framework on which elements such as paragraphs and images can be organized in a rational way. In the context of web design CSS-based grids serve not only to reduce the complexity on content-level, but they also provide an abstraction layer for growingly complex CSS layouts. Over the past years several grid systems have been developed that facilitate responsive layouts on desktops, tablets and mobile phones by means of media queries.

The Responive Grid control is a layout which positions its child controls in a 12 column flow layout. Its children can be specified to take on a variable amount of columns depending on available screen size. With this control it is possible to achieve flexible layouts and line-breaks for large-, medium- and small-sized screens, such as desktop, tablet, and mobile. The Grid control's width can be percentage or pixel-based and the spacing between its columns can be set to various pre-defined values.

The Grid can be placed in almost any container and be set to take on its width. The spacing between its columns can be set to 0, 1rem, or 2rem. The columns take on the remaining space. Alternatively, the grid can be set to a fixed pixel width by means of CSS.

We recommend to use the Grid as the default container control for the SAP Web IDE. It is built into the Wizard, used to lay out this documentation, and it also should be used for all right panes.

Do

  • Do use the Grid
  • When placing a control inside of the Grid, set its width to 100%. The Grid then will take care of its width.
  • Check if the control looks nice on all resolutions
  • In case a control is cropped, consider using a bigger width property for its grid container
  • Use the parameters for vertical spacings between the grid containers

Don't

  • Do not stack horizontal and vertical layouts
  • Do not overwrite horizontal spacings of grid containers since this might break the layout
