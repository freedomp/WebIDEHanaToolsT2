Visual Design Foundations

Colors

The color balance of the SAP Web IDE refers to the recommended mixture of color and contrast of any SAP application.

It creates a visual rhythm throughout the application that helps drawing the user's attention to the most important information and functions. It further promotes a distinct and consistent look and feel throughout the entire integrated development environment.

The colors scheme of the SAP Web IDE is inspired by SAP Fiori. Several major color values have been derived from existing Fiori colors, are re-used, or show up in different contexts, E.g. the SAP Web IDE highlight color is 10% darker than its Fiori equivalent and the main background color of the SAP Web IDE can be found in the Fiori footer bar.

The color scheme for the Fiori visual design

The derived color scheme for the SAP Web IDE

This derivation of color values makes it possible to show two individual visual design languages in one user interface. When shown together, as done in the Preview or the WYSIWYG editor for example, they are not mixed up, yet look related.

Background Colors

For its elements the SAP Web IDE uses a slate gray as background color. It can be used for side bars, tiles, icons and other foreground elements as well. To balance the slate gray and the blue highlight and branding colors, the SAP Web IDE makes use of a very bright beige color. This background counterpart can be used for containers in the editor area, side panes, and tiles.

Accent Colors

Accent colors can be used to distinguish or decorate elements. The majority of colors stem from the color pallete of the Fiori visual design. The code editor makes heavy use of these colors. They are a vivid contribution to the overall UI and should be used sparingly. As a rule of thumb these colors should occur in the main area and not the side panes or sidebars.

Greyscale Colors

Greyscale areas play an important role in the user interface of the SAP Web IDE. They minimize the risk of over-stimulation and foster simplicity. White and light grays are mainly used for background areas. The darker grays are used for texts.

Editor Themes

The editor of the SAP Web IDE comes in four color themes: Cumulus, Basement, Flower Power, and Morlock. Cumulus is a very bright theme with a bright blue syntax highlighting. Basement and Morlock are both darker; Basement has a reduced luminosity whereas Morlock is the more colorful and vivid theme. Flower Power makes heavy use of the Fiori accent colors and thus is the most vibrant one. Cumulus is shipped as the default editor theme of the SAP Web IDE and fulfills SAP's product standards in regards of contrast.

We plan to extend the theming capabilities of the SAP Web IDE from the editor to all the user interface elements. Doing so, not only the editor will become themable, but changes in the theme will have effect on the colors of panes, bars, icons as well.

Do

  • Use the provided less parameters wherever possible. This helps making the whole UI of the SAP Web IDE ready for theming.
  • Keep the main area (i.e. the editor) colorful.
  • Keep secondary areas (panes, bars etc.) monochromatic.

Don't

  • Do not use Accent colors in the toolbars, sidebars, or the side panes.
  • Do not use gradients.
