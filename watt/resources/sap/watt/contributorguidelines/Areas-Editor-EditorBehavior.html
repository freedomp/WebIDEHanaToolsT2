Tabs and Representing Views

In the SAP Web IDE, opened files are displayed as tabs in the main area. Depending on its type, each file can have varying representations of their content. These representations are called Views (?).

For example, a certain XML file can be represented by a Code view (which shows the actual XML code as text), by a Design view (which shows the UI elements contained in the XML in an WYSIWYG interface), or by a Data Binding view (which shows an interface to connect UI elements to data sources). These views are shown as additional tabs at the bottom of each file (?).

Links

To connect different files and views to each other, the SAP Web IDE makes use of Hyperlinks. Selecting a link in a view, the related file is opened in the requested views.

Pop-Out / Dock-out

To show two views of one file side-by-side, a currently unselected view can be popped-out to the side of the selected view via the contextual menu of its tab. The tabs at the bottom then are displayed as one merged tab. Popped-out views can be hidden again via the contextual menu of the merged tab. The tabs then again are displayed as separate tabs.

Split mode

To display not only views but also files side-by-side, the entire editor can be split in two areas. The split mode can be triggered via the contextual menu of file in the repository browser.
