• SAP Web IDE — Design Guidelines

Visual Design Foundations

Numerology

As done in the visual design for Fiori, the SAP Web IDE embraced a measuring system using the rem unit, which was introduced with CSS3. In both contexts, a base size of 16 pixels defines 1 rem. The base of 16 pixels allows for maintaining a consistent visual design for the SAP Web IDE. In addition, this common foundation makes it easier to show two individual visual design languages in one user interface. When the Fiori and SAP Web IDE visual designs are shown together, as done in the Preview or the WYSIWYG editor for example, they keep a related look and feel by means of similar spacings.

The interface elements of the SAP Web IDE possess a base height of either 2 rem (32 pixels) or 1.5 rem (24 pixels). And also their spacing and measurements are based on 16 pixels. This “numerology” always should be considered when designing user interfaces and plugins that integrate well into the existing style of the SAP Web IDE.

This incomplete list shows values for sizes, spacing and measurements in the SAP Web IDE derived from the base of 16 pixels:

0.25 rem (4 pixels)

  • Width of the splitter, separating panes and editor
  • Upper margin of tabs
  • Spacing surrounding tooltips

0.5 rem (8 pixels)

  • Spacing surrounding panels
  • Spacing surrounding emphasized containers in the panels
  • Spacing between buttons
  • Width of all scrollbars

1 rem (16 pixels)

  • Default gutter width of the Grid (used in the wizard)
  • Icon size
  • Spacing between groups in the toolbar
  • Spacing surrounding dialogs
  • Spacing surrounding pop-ups

1.5 rem (24 pixels)

  • Default height of small interface elements
  • Height of menu entries
  • Height of repository browser tree items

2 rem (32 pixels)

  • Default height of large interface elements
  • Height of the menu bar
  • Height of the toolbar
  • Height of the tab bar
  • Height of the heading for the right pane
  • Optional gutter width of the Grid
  • Large icon size, i.e. used in tiles and the control pallet

3 rem (48 pixels)

  • Height and width of the button in the sidebars
  • Largest font size, e.g. used in the wizard

Do

  • Base spacings, paddings, and margins on 16 pixels
  • Re-use spacings of existing interface elements
  • Re-use exsitig CSS classes for spacings

Don't

  • Do not invent custom spacings, paddings, or margins
  • Do not invent custom font sizes
